概要总览

我们要做什么

基于 N3 Lite 现有本地蓝牙功能，构建数据上云通道。将负载均衡监控、充电控制、充电记录、告警等功能全部接入云平台，实现远程可见、可查、可配。

1.0 版本聚焦于打通数据通道，验证稳定性。后台管理系统先行，用户侧 App 是后续版本的事。

核心目标
  • 设备上线可见: N3 Lite 连上云，后台看到设备状态和下挂桩拓扑
  • 数据长期保存: 充电记录、能耗数据不再受限于 ESP32 Flash 容量
  • 远程监控诊断: 出门也能看到实时数据，设备故障远程可见
  • 远程 OTA: 新功能和 Bug 修复不再需要上门刷机
  • 远程配置: DLB 参数可从云端调整，无需蓝牙近场操作

一、N3 Lite 产品现状
我们的起点 — 本地蓝牙充电管理

N3 Lite 已在本地实现了完整的充电管理功能。了解现状，才能明确上云要做什么。

产品定位

1.1 产品定位

Neuron III Lite 是旭衡电子推出的智能充电管理系统，专为家庭场景设计。安装在家庭配电箱 DIN 导轨上，通过 PLC 电力线通信管理最多 2 台 ATP III 充电桩。用户通过手机蓝牙 App 进行本地控制和监控。

目前产品定位: 本地化部署，无需云端连接，注重用户数据隐私保护。

产品套装

Set A: 单桩家庭
1× N3 Lite + 1× ATP III 充电桩
Set B: 双桩家庭
1× N3 Lite + 2× ATP III 充电桩（DLB 共享电流）
功能清单

1.2 现有功能清单（蓝牙本地版）

以下按照 App 的三个 Tab 页面结构整理。

Tab 1: 充电桩（首页）— 负载均衡监控 + 设备管理

功能 1: 设备连接与管理

  • BLE 搜索附近 N3 Lite（按序列号显示）
  • 连接 / 断开 N3 Lite
  • 通过 PLC 搜索附近充电桩
  • 添加充电桩到 N3 Lite（最多 2 台）
  • 删除充电桩（右划卡片删除）
  • 设备卡片信息: ID 编号、连接状态（在线/离线/充电中/暂停中）、实时电流

功能 2: 负载均衡监控（DLB 核心看板）

  • 用电拓扑图: 直观显示充电桩、家庭负载、总入户电流的关系
  • 实时读数: 家庭负载电流值、各充电桩电流值、入户总电流
  • 最大入户电流设置: 6 个挡位 — 20A / 25A / 32A / 40A / 50A / 63A
    • 设置方式: App 点击切换 或 设备 SET 按键
    • 工作原理: 系统根据设定值做 DLB 控制，家庭其他电器用电增加时自动降低充电功率，防止跳闸
警告: 最大入户电流设置必须与家庭总进线空气开关规格匹配，设置不当可能导致跳闸或安全隐患。

功能 3: 充电控制

  • 枪连接检测: 自动检测充电枪是否正确插入车辆
  • 开始充电: 枪连接后蓝色按钮激活，点击启动
  • 实时充电监控: 充电电流动画、充电时长、已充电量(kWh)
  • 结束充电: 红色按钮停止
  • 充电结束自动生成单次记录: 设备ID、开始/结束时间、时长、电量

功能 4: 充电中安全锁定

当任意设备处于充电状态时，系统自动锁定以下操作:

  • 禁止修改最大入户电流 — 防止 DLB 计算失误导致跳闸
  • 禁止添加或删除设备 — 防止拓扑变动影响控制稳定性
  • 禁止 OTA 升级 — 防止重启中断充电控制进程

Tab 2: 记录 — 历史充电记录

功能 5: 充电记录查询

  • 显示历史充电次数和总充电量(kWh)
  • 记录列表: 设备ID、充电时间、时长、电量
  • 日期筛选: 选择不同日期查看对应时间段记录
  • 数据存储在 N3 Lite 本地 Flash

Tab 3: 我的 — 系统设置

功能 6: 系统语言

  • 支持: 简体中文、繁体中文、英文

功能 7: OTA 固件升级（BLE 本地）

  • 显示当前固件版本
  • 从手机本地选择 .bin 固件包（格式: N3Lite-x.x.x.bin）
  • 通过蓝牙传输升级，显示进度
  • 升级完成设备自动重启
  • 重连后可确认新版本
二、上云价值分析
为什么要上云 — 痛点与目标

分析现有痛点，明确上云要解决的问题和预期收益。

痛点分析

2.1 现状痛点

痛点具体影响业务代价
蓝牙距离限制用户必须在设备旁（约10米内）才能操作和查看出门后完全失联，用户不知道充电是否完成
数据存储有限充电记录存 ESP32 Flash，容量有限长期充电记录无法保留，用户无法查看历史趋势
无法远程诊断设备故障只能通过 SYS 灯红色提示每次售后都要上门，成本高、响应慢
OTA 依赖蓝牙固件升级需要用户在设备旁操作新功能推送覆盖率低，Bug 修复周期长
无全局视图无法掌握已售设备的运行状况产品迭代缺乏数据支撑，无法主动发现问题
无集中管理每台设备独立运行，互不关联无法批量配置、批量升级、统一监控
核心问题: N3 Lite 卖出去就是一座孤岛。用户出门失联、售后全靠上门、产品迭代没有数据 — 所有问题的根源都是没有云端连接。
上云目标

2.2 上云目标

对平台运营方（1.0 版本核心受益方）

  • 设备全局视图 — 所有已售 N3 Lite 在线状态、运行数据一目了然
  • 远程诊断 — 不上门就能看到设备状态和数据，快速定位问题
  • 远程 OTA — 一键批量推送固件，覆盖率高，成本低
  • 数据驱动 — 充电量统计、故障率分析，支撑产品迭代决策
  • 远程 DLB 配置 — 售后人员可远程调整客户的 DLB 参数

对用户（后续版本受益）

  • 出门也能看 — 充电状态、充电记录随时可查
  • 数据永久保存 — 充电记录上云，不受设备存储限制
  • 告警即时推送 — 设备故障第一时间推送到手机
  • 远程控制（预留） — 未来可通过 App 远程启停充电
1.0 版本定位: 后台管理系统先行 — 先把数据通道打通，验证稳定性。确保 N3 Lite 能稳定连接云平台、数据准确上报、远程 OTA 可靠运行。用户侧 App 是后续版本的事。
三、功能需求 — 逐项云化设计
从本地蓝牙到云端 — 每项功能怎么云化

逐一对照 N3 Lite 本地现状，明确上云后的需求和后台页面设计。

功能需求

3.1 设备注册与在线管理 新增

本地现状

  • 本地没有"注册"概念。N3 Lite 通过蓝牙被手机发现，连接即用
  • 平台侧无法知道设备卖出去后的状态

上云需求

  • N3 Lite 通电后自动通过 WiFi 连接云平台，完成设备注册
  • 首次连接: 自动注册（序列号、型号、固件版本）
  • 后续连接: 自动识别已有设备，更新在线状态
  • 心跳保活: 定时上报心跳，云平台实时掌握在线/离线状态
  • 离线检测: 超过一定时间未收到心跳，标记为离线

后台页面需求

  • 设备列表: 序列号、型号、固件版本、在线状态（在线/离线/故障）、最后心跳时间
  • 设备总览: 总设备数、在线数、离线数、故障数
  • 搜索/筛选: 按序列号搜索、按状态筛选
功能需求

3.2 设备拓扑管理 新增

本地现状

  • N3 Lite 通过 PLC 搜索并绑定充电桩，拓扑信息只存在本地
  • App 首页显示拓扑图（N3 Lite + 下挂桩）
  • 支持添加/删除充电桩

上云需求

  • N3 Lite 连接云平台后，自动上报当前绑定的充电桩拓扑
  • 拓扑信息包括: 下挂桩的序列号、型号、枪数、连接状态
  • 桩增减时自动更新拓扑（添加或删除充电桩后重新上报）
  • 云平台以设备树形式展示完整拓扑

后台页面需求

设备树视图示例:

N3L-2026-00001 (N3 Lite) [在线]
  ├── ATP-A001 (ATP III 7kW) 枪1 [可用] / 枪2 [充电中]
  └── ATP-B002 (ATP III 7kW) 枪1 [可用] / 枪2 [离线]
  • 点击任一节点可查看详细信息
为什么需要拓扑上云？
售后人员需要远程了解客户家里的设备组成。"你家 N3 Lite 下面挂了几根桩？哪根桩离线了？" — 有了拓扑数据，远程就能判断，不用上门。
功能需求

3.3 CT 能耗监测 本地有 → 上云增强

本地现状

  • CT 互感器实时采集家庭进线电流
  • App 负载均衡监控页显示: 家庭负载电流、各桩电流、入户总电流
  • 电能监测: 电压、电流、功率
  • 数据只能蓝牙近场查看，无法保存历史

上云需求

  • N3 Lite 定时将 CT 数据上报云平台（电流、电压、功率、温度）
  • 云端存储时序数据，支持历史查询
  • 数据可视化: 实时 + 历史趋势

后台页面需求

  • 实时数据卡片: 家庭进线电流(A)、总功率(W)、电压(V)、设备温度(°C)
  • 历史趋势图: 可选时间范围（1 小时 / 24 小时 / 7 天 / 30 天）
  • 用电统计: 日 / 周 / 月累计用电量(kWh)
  • 支持按设备筛选
功能需求

3.4 DLB 动态负载均衡 本地有 → 上云可配置

本地现状

  • DLB 核心功能: 根据最大入户电流设定值，实时计算剩余可用电流，动态分配给各充电桩
  • 分配模式: 默认平均分配
  • 防跳闸保护: 家庭其他电器用电增加时，自动降低充电功率
  • 6 级最大入户电流设置: 20A / 25A / 32A / 40A / 50A / 63A
  • 设置方式: App 蓝牙 或 设备 SET 按键
  • 充电中禁止修改设置（安全锁定）

上云需求

  • 上行: N3 Lite 定时上报 DLB 分配状态（各桩分配电流、总可用容量、分配模式）
  • 下行: 云端可远程修改 DLB 配置（最大入户电流、分配模式）
  • DLB 算法仍在 N3 Lite 本地执行，不依赖云端
  • 云端角色: 监控 + 远程配置，不参与实时控制

后台页面需求

  • DLB 实时面板: 各桩电流分配（柱状图/环形图）、总容量/已用/可用
  • DLB 配置: 最大入户电流（6 挡选择）、分配模式 — 修改后即时下发
  • DLB 历史: 电流分配趋势图
  • 防跳闸事件记录: 触发时间、总电流、阈值、各桩电流调整详情
为什么 DLB 算法不放云端？
DLB 是安全关键功能。N3 Lite 断网时必须能独立工作 — 家庭不能因为网络问题而跳闸。DLB 必须在本地实时执行（毫秒级响应），云端网络延迟无法满足。云端的角色是"远程监控 + 远程调参"，不是"远程控制"。
为什么售后需要远程改 DLB 配置？
用户可能不懂怎么设置最大入户电流，或者设置错了导致频繁跳闸。售后人员远程就能帮用户调整，不用上门。
功能需求

3.5 充电控制与实时监控 本地有 → 上云记录

本地现状

  • 枪连接检测: 自动检测充电枪是否正确插入
  • 开始充电: 枪连接后 App 蓝色按钮激活
  • 实时监控: 充电电流动画、充电时长、已充电量(kWh)
  • 结束充电: App 红色按钮
  • 充电结束自动生成记录
  • 充电中安全锁定: 禁止改 DLB 设置、禁止增删设备、禁止 OTA

上云需求（1.0 版本）

  • 充电事件上报: 充电开始/结束时，N3 Lite 上报充电事件（桩ID、开始/结束时间、电表读数）
  • 充电中数据上报: 定时上报实时充电数据（功率、电流、已充电量）
  • 云端记录: 所有充电记录持久化到云端数据库
  • 桩状态同步: 各桩状态（可用/充电中/故障/离线）实时同步到云端
  • 远程启停充电: 预留，1.0 版本暂不实现

后台页面需求

  • 设备状态: 在设备详情页显示各桩当前状态（可用/充电中/故障）
  • 正在充电: 如果有桩在充电，显示实时充电功率、时长、已充电量
  • 充电记录列表: 设备、桩号、开始/结束时间、时长、电量(kWh)
  • 充电详情: 功率曲线图（按充电过程中的数据绘制）
  • 充电统计: 按设备 / 按天 / 按月 的充电量汇总
预留功能 — 远程启停充电: 云端远程启动/停止充电的能力在架构中预留，但 1.0 版本不开放。1.0 优先打通数据通道，充电控制仍由用户在本地通过蓝牙 App 操作。待用户版 App 开发后启用。
功能需求

3.6 充电记录上云 本地有 → 上云持久化

本地现状

  • 充电记录存在 N3 Lite 本地 Flash
  • App "记录" Tab 显示: 历史充电次数、总充电量
  • 每条记录: 设备ID、充电时间、时长、电量(kWh)
  • 支持日期筛选
  • Flash 容量有限，无法长期保留所有记录

上云需求

  • 每次充电开始/结束，N3 Lite 自动上报到云平台
  • 云端永久保存所有充电记录
  • 充电记录是关键业务数据 — 必须保障零丢失（断网缓存、恢复后补发）
  • 支持导出（预留）

后台页面需求

  • 充电记录列表: 设备、桩号、开始时间、结束时间、时长、电量(kWh)
  • 筛选: 按设备、按日期范围
  • 统计看板:
    • 全部设备: 总充电次数、总充电量
    • 按设备: 单台设备的充电次数、充电量
    • 按时间: 日/周/月充电量趋势图
充电记录为什么必须零丢失？
充电记录是产品核心数据。对平台: 是分析产品使用情况的基础。对用户(未来): 是计算电费、证明充电行为的依据。丢了一条记录 = 用户质疑产品可靠性。
功能需求

3.7 OTA 远程升级 BLE本地有 → 上云远程

本地现状

  • OTA 通过蓝牙完成: 用户在 App "我的" Tab 选择本地 .bin 文件上传
  • 升级过程需保持蓝牙连接，不能切屏
  • 升级完成设备自动重启
  • 固件格式: N3Lite-x.x.x.bin
  • 充电中禁止 OTA

上云需求

  • N3 Lite OTA: 云平台下发固件下载地址，N3 Lite 通过 WiFi 自行下载安装
  • 下挂桩 OTA: 云平台通知 N3 Lite，N3 Lite 通过 PLC 中转给充电桩
  • 固件包管理: 上传、版本管理、存储
  • 升级策略: 单台 / 批量 / 灰度（先 10% → 观察 → 全量）
  • 安全: 固件包 SHA256 校验，失败自动回滚（双分区机制）
  • 充电中不允许 OTA（与本地逻辑一致）

后台页面需求

  • 固件管理: 版本列表（版本号、上传时间、文件大小、适用设备）、上传新版本
  • 创建升级任务: 选择固件版本 → 选择目标设备（单台/批量/按条件筛选）→ 设置计划时间
  • 升级进度: 每台设备的状态（等待/下载中/安装中/成功/失败/已回滚）
  • 升级记录: 任务历史、成功率统计
远程 OTA 相比蓝牙 OTA 的核心优势？
1. 覆盖率: 蓝牙 OTA 需要用户主动操作，大部分用户不会做 → 新版本覆盖率极低。远程 OTA 一键推送。
2. 效率: 蓝牙一次升级一台，几百台设备要几百次操作。远程批量升级，一次搞定。
3. 速度: 蓝牙传输慢（BLE 带宽低），WiFi 下载快几十倍。
4. 紧急修复: 发现安全漏洞或严重 Bug，远程 OTA 可以小时级修复，蓝牙 OTA 可能要几周。
功能需求

3.8 告警管理 本地有 → 上云增强

本地现状

  • SYS 灯红色长亮: 告警提示（需重启，重启后仍红色需联系技术支持）
  • 蓝牙 App 可查看设备状态
  • 无告警推送能力（蓝牙断开后无法通知用户）

上云需求

  • 设备告警上报: 故障事件实时上报云平台
  • 设备离线告警: 心跳超时触发离线告警
  • DLB 防跳闸事件: 过载保护触发时上报
  • 告警分级:
级别触发场景通知方式
严重接地故障、过温保护立即通知
重要过流/过压/欠压、设备故障及时通知
一般设备离线、DLB 限流记录，汇总通知
信息OTA 升级结果仅记录
  • 通知方式（1.0）: 后台页面展示，预留邮件/Webhook

后台页面需求

  • 告警列表: 时间、设备、级别（颜色标记）、类型、描述、是否已确认
  • 告警统计: 按级别/类型/设备分组
  • 未处理告警数: 在导航栏/首页显示红色角标
  • 告警确认: 操作员可标记已处理

功能需求到后端模块的映射

模块划分

4.1 模块划分与功能映射

基于第三章的功能需求，后端划分为以下模块。每个模块职责单一、边界清晰，未来可独立拆分为微服务。

neuron-protocol
设备接入网关
N3 Lite 的 WebSocket 长连接入口。负责协议解析、心跳保活、消息收发，是所有设备数据进入云平台的唯一通道。
neuron-device
设备管理
设备注册、在线状态、拓扑关系、DLB 配置管理。承接 3.1 设备注册、3.2 拓扑管理、3.4 DLB 配置。
neuron-energy
能耗数据
CT 采集数据的存储与可视化，时序数据查询，用电统计。承接 3.3 CT 能耗监测。
neuron-charge
充电管理
充电事件处理、充电记录持久化、充电统计。承接 3.5 充电控制与监控、3.6 充电记录上云。远程启停充电预留。
neuron-ota
OTA 升级
固件包管理、升级任务创建与调度、升级进度跟踪。承接 3.7 OTA 远程升级。
neuron-alert
告警管理
告警规则引擎、告警事件处理、告警通知、告警确认与统计。承接 3.8 告警管理。
neuron-api
前端 API 聚合层
所有后台管理页面的 REST API 统一入口。聚合各业务模块的数据，向前端提供接口。承接 3.9 首页仪表盘 及所有后台页面的接口需求。
neuron-system
系统管理
用户、角色、权限、菜单、字典等基础管理功能。所有模块的公共基础设施。

neuron-api 的定位: 它不包含业务逻辑，只做接口聚合。前端所有页面的 REST 请求都走 neuron-api，由它调用各业务模块获取数据。好处: 前端只对接一个模块，业务模块之间保持解耦。
neuron-protocol 的定位: 它是设备数据的入口，但不处理业务逻辑。收到设备消息后，投递给对应的业务模块处理。好处: 协议层和业务层解耦，更换协议不影响业务。
