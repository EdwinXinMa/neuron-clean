一、设计出发点
以设备为基础，以自身使用为方向

先理解设备，再从我们自己的工作场景出发设计。

BACKGROUND

1.1 N3 Lite 背景

N3 Lite 是一台家用智能充电网关，出货后自主运行。设备通过 PLC 管理下挂充电桩，通过 CT 传感器实时读取家庭电流，自动进行动态负载调节（DLM）。设备一旦部署，便在客户侧独立运行，不需要我们的日常介入。

这决定了我们与设备的关系：我们是设备的出货方与守护者，而不是使用者。我们关心的是出货设备的整体运行状况。

基于这个角色，我们的工作天然分为两种状态：

大多数时候 — 被动监控
设备正常运行，无需干预。我们只需要知道整体情况：在线多少台、有没有故障、设备分布在哪里。靠看。
少数时候 — 主动处理
出现告警，或客户反馈问题。需要快速定位这台设备，一屏看清全部状态，必要时远程干预。靠查与操作。

这两种工作状态，直接决定了平台的设计方向。

1.2 核心场景

基于我们的角色，平台围绕两个核心场景展开：

总览监控：
设备出货总量、运行中、离线、故障等数量，以及分布区域
定位排查：
按序列号找到这台设备 → 一屏看清设备状态、家庭电力数据、下挂充电桩状态 → 必要时远程修改参数或推送升级

1.3 设计思路

大多数时候我们在看，少数时候才需要操作。所以：

以看为主，操作为辅。打开即是信息，操作内嵌在数据旁边，需要时才触发。导航从简，几个视图一目了然，不需要层层菜单。视线始终留在数据上，不被导航分散。

LAYOUT

2.1 布局结构

顶部导航，内容区全宽展示，地图和分屏布局充分铺开。

  1. 导航在顶部 — 我们日常只需要在几个固定视图之间切换，顶部一行导航栏即可完成，视线始终停留在内容区，不会被侧边菜单分散注意力。
  2. 全宽内容区 — 设备地图需要大面积铺开才能看清设备分布，全宽布局让地图、表格、数据卡片都有充足的展示空间，打开就能一屏掌握全局。
  3. 左右分屏 — 排查某台设备时，左侧列表快速定位，右侧立即展开详情，不需要跳转页面。找到设备和查看信息在同一屏完成，缩短定位到处理的路径。

3.3 告警中心

服务于：定位排查
设备出现异常时，需要一个地方集中查看和处理。哪些问题还没处理、哪些已经跟进，不遗漏。

告警分级

级别颜色触发条件处理优先级
严重🔴 红色接地故障、过温保护立即处理
重要🟠 橙色过流、过压、欠压、设备故障码当日处理
一般🔵 蓝色设备离线、DLM 限流频繁关注跟踪
信息⚫ 灰色OTA 升级结果、设备首次激活记录即可

页面设计

  • 默认排序：未处理优先，同状态按级别从高到低排列
  • 筛选条件：告警级别 / 处理状态（未处理/已处理）/ 时间范围
  • 处理操作：标记已处理 + 填写处理备注
  • 跳转联动：点击告警条目中的设备序列号 → 进入设备视图并展开该设备

全局告警通知

页面内醒目提示为主。导航栏告警图标实时显示未处理告警数（红色角标），任何视图下都可见。

3.4 操作日志

服务于：定位排查
对设备做过什么，都应该有据可查。升级了哪个版本、改过哪个参数、谁操作的、结果如何，每一步操作都可追溯。

两个层面的操作记录

设备维度（在设备视图内）
在设备视图右侧详情的"操作日志" Tab 中，展示这台设备的历史操作记录。快速了解这台设备被操作过什么。
全局维度（操作日志视图）
顶部导航"操作日志"入口，展示所有设备的操作汇总。管理者可追溯谁在什么时间对哪台设备做了什么操作。

3.5 设置

所有低频的管理操作集中在设置中，不占用主导航空间。

固件库
上传 .bin 固件文件，维护版本列表。设备视图触发 OTA 升级时，从这里选择目标版本。

字段：版本号、适用型号、文件大小、SHA256 校验值、上传时间、变更说明

文件存储：上传到对象存储（Minio），数据库只存元数据和下载地址。设备 OTA 时云平台下发下载 URL，设备自行下载安装。
账号管理
用户账户的维护。

功能：账号列表、新增账号、禁用账号、重置密码
设备台账
出货时人工录入设备基础信息，作为设备档案的补充。设备自动注册时，台账信息为可选补录。

字段：序列号、设备型号、生产批次、经销商、出货日期、位置坐标

录入方式：手动录入 / Excel 批量导入

4.1 实时更新

前端通过 WebSocket 接收后端推送，无需手动刷新页面。涉及实时更新的场景：

#事件前端响应影响范围
1设备状态变化（上线 / 离线）总览数字卡片更新，地图点颜色同步，设备列表状态更新总览 + 设备视图
2告警状态变化（触发 / 处理）导航栏角标实时增减，总览告警动态同步全局导航 + 总览
其他数据（设备详情、家庭电力数据等）通过 HTTP 请求拉取，点击设备时加载一次，不持续推送。

4.3 视图联动跳转

各视图之间有明确的联动关系，减少用户在导航栏来回切换的次数：

总览大屏
  ├── 点击地图设备点 → 设备视图（自动展开该设备详情）
  ├── 点击数字卡片（如"故障 3"）→ 设备视图（自动筛选对应状态）
  └── 点击告警动态条目 → 设备视图（自动展开该设备详情）

告警中心
  └── 点击告警条目的设备序列号 → 设备视图（自动展开该设备详情）

设备视图（左侧列表）
  └── 支持直接搜索序列号快速定位设备
核心设计思路：用户不需要记住"这台设备在哪个页面能找到"。无论从总览、告警还是操作日志，点击设备相关信息都会直接定位到设备视图并展开该设备的详情。
