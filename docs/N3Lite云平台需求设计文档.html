一、项目背景
围绕N3 Lite 产品，设计优先服务于我们自己的云端管理平台

1.1 N3 Lite

专为家庭场景设计。安装在家庭配电箱 DIN 导轨上，通过 PLC 电力线通信管理最多 2 台 ATP III 充电桩的智能网关

当前工作方式：用户通过手机蓝牙 App 在本地控制 — 启停充电、查看状态、调整 DLM 参数。一切都在本地完成，无需联网。

1.2 云平台

优先服务于 我们自己 的 N3Lite 云端管理平台

未来的实现：我们可通过云平台 总览所有设备的实时状态、区域分布、告警信息，及时推送告警信息。还可以配合用户对N3Lite进行远程升级、配置。

CONNECT

3.1 发货登记

为什么需要发货登记？
客户打电话来报名字，我们要能立刻查到他的设备。如果没有发货时绑定客户信息，设备上线了也不知道是谁的。发货登记把设备序列号和客户信息关联起来，是后续一切运维的基础。

功能要求

  • 发货前人工录入：设备序列号 + 客户姓名 + 客户电话 + 收货地址
  • 录入后设备状态标为 已发货/未激活
  • 支持批量录入（Excel 导入）
  • 发货记录列表：可按序列号、客户姓名、电话、发货日期搜索

后台页面：设备发货登记

设备发货登记页面
  • 列表字段：序列号、设备型号、客户姓名、客户电话、收货地址、发货日期、当前状态
  • 操作：新增、编辑、删除、批量导入（Excel）、导出
  • 状态标签：未激活 在线 离线 故障
CONNECT

3.2 自动激活与心跳

为什么需要自动激活？
客户收到设备通电联网后，不需要任何额外操作就能在云端"亮起来"。手动激活流程越复杂，客户越不会配合，设备就越可能一直是"未激活"状态。

激活流程

1
N3 Lite 通电联网 — 连接家庭 WiFi 后自动连接云平台 WebSocket
2
上报身份信息 — 序列号、设备型号、固件版本
3
云平台匹配 — 用序列号匹配发货登记记录，状态从"已发货/未激活"变为 在线
4
心跳保活 — 每分钟发送心跳，超时（如3分钟无心跳）标为 离线
设计原则：未在发货登记中的序列号也允许接入（记录为"未登记设备"），避免因漏登记导致设备无法上线。后台可以后补客户信息。
CONNECT

3.3 下挂桩拓扑

为什么需要拓扑上报？
客户说"充电桩坏了"，我们得知道他有几台桩、每台桩什么状态。N3 Lite 是网关，它下面挂着的 ATP III 桩才是直接给车充电的设备。不看拓扑，就无法定位问题出在网关还是桩。

功能要求

  • N3 Lite 启动时自动上报下挂 ATP III 桩信息：桩序列号、桩型号、桩当前状态
  • 拓扑变更时（添加/删除桩）重新上报
  • 云平台维护 N3 Lite ↔ ATP III 的父子关系

后台展示

在设备详情页中，以树形结构展示：

N3 Lite (SN: NL-20260301-0001) [在线]
  ├── ATP III-11.5 (SN: AT-20260301-0001) [可用]
  └── ATP III-11.5 (SN: AT-20260301-0002) [充电中]

4.2 诊断页

这是最实用的页面。客户打电话来报个名字或电话，平台立刻查到设备，一屏看完全貌。直接减少上门次数。
为什么需要诊断页？
客户打电话说"我家充电桩不工作了"。平台需要在 10 秒内找到他的设备，然后判断：是离线了还是有故障码？DLM 设置对不对？桩是什么状态？这些信息分散在不同页面就没用了 — 必须一屏呈现。

搜索方式

  • 模糊搜索：支持按客户姓名 / 客户电话 / 设备序列号搜索
  • 搜索即显结果，无需精确匹配

设备全貌（一屏展示）

信息项说明

在线状态在线 / 离线 + 最后心跳时间

当前错误码有无活跃故障，错误码含义
DLM 设置当前断路器档位（如 32A）
下挂桩状态每台桩的序列号、型号、当前状态（空闲/充电中/故障/离线）

4.3 告警监控

为什么需要主动告警？
不等客户打电话。设备故障了我们比客户先知道，主动联系客户解决，这才是设备商该有的服务水平。离线告警更重要 — 设备不响应可能是网络问题，也可能是硬件故障，越早发现越好。

告警分级

级别标记触发条件处理优先级
严重红色接地故障、过温保护立即处理
重要橙色过流、过压、欠压、设备故障码当日处理
一般蓝色设备离线（心跳超时）、DLM 限流频繁关注跟踪
信息灰色OTA 升级结果、设备首次激活记录即可

后台页面：告警列表

告警列表页面
  • 列表字段：告警时间、设备序列号、客户姓名、告警级别、告警类型、告警描述、处理状态
  • 筛选：按级别、按类型、按处理状态、按时间范围
  • 操作：标记已处理（填写处理备注）、跳转到设备诊断页
  • 默认排序：未处理在前，按级别从高到低

导航栏告警角标

导航栏红色角标：未处理告警总数实时显示在导航栏告警图标右上角（红色数字角标）。任何页面都能一眼看到有没有待处理的告警。点击直接进入告警列表页。

5.1 远程 OTA 升级

固件包管理

功能说明
上传固件包上传 .bin 文件，填写版本号、变更说明、适用设备类型（N3 Lite / ATP III）
版本管理版本列表：版本号、上传时间、文件大小、适用设备、变更说明、SHA256 校验值
安全校验SHA256 校验，设备端验证通过才安装，校验失败自动拒绝

升级策略

升级方式

  • 单台升级：指定单台设备推送
  • 批量升级：选择多台设备批量推送
  • 灰度升级：先 10% 设备 → 观察 24h → 无异常全量推送

安全机制

  • 充电中不允许 OTA — 等充电结束再升级
  • 双分区机制 — 失败自动回滚到上一版本
  • SHA256 校验 — 防止损坏固件被安装

OTA 传输路径

A
N3 Lite OTA：云平台下发固件下载地址 → N3 Lite 通过 WiFi 自行下载安装
B
下挂桩 OTA：云平台通知 N3 Lite → N3 Lite 下载固件 → 通过 PLC 中转给 ATP III 充电桩

升级进度跟踪

OTA 升级任务页面
  • 任务概览：目标版本、目标设备数、成功/失败/进行中/等待中数量
  • 每台设备状态：等待 下载中 安装中 成功 失败 已回滚
  • 失败设备可查看失败原因，支持一键重试
  • 升级历史记录：每次升级任务的详细记录

5.2 远程 DLM 配置

为什么需要远程调 DLM？
核心功能！客户买回去自己设 DLM 断路器档位。设高了 → 老跳闸；设低了 → 充电慢。目前只能客户用蓝牙 App 改，很多客户搞不定。现在后台可远程一键就能帮调。

功能要求

功能说明
可配置参数最大入户电流档位（断路器额定值）：20 / 25 / 32 / 40 / 50 / 63A
下发方式在诊断页或设备详情页选择档位 → 下发到 N3 Lite
生效确认N3 Lite 收到后回复确认，云端显示"已生效"
充电中保护充电中禁止修改 DLM 设置（与本地逻辑一致），界面提示"充电中，请稍后修改"
重要设计原则：DLM 算法仍在 N3 Lite 本地执行，云端只是远程调参，不参与实时控制。这保证了即使断网，DLM 仍然正常工作。云平台的角色是"远程遥控器"，不是"远程大脑"。
六、数据来源
两个源头

所有数据要么设备主动上报，要么人工录入。没有第三个来源。

DATA SOURCE

6.1 设备上报 + 人工录入

数据源一：设备上报（N3 Lite → 云平台）

N3 Lite 通过 WebSocket 连接云平台，主动上报以下 6 种数据：

#数据类型内容触发时机频率
1身份注册序列号、型号、固件版本首次通电联网时一次
2心跳"我还活着"周期性每分钟
3下挂桩拓扑桩序列号、桩型号、桩状态启动时 + 拓扑变更时事件驱动
4故障上报错误码、故障类型故障发生时事件驱动
5DLM 状态断路器档位、各桩分配电流参数变更时事件驱动
6OTA 结果升级成功 / 失败 + 原因OTA 完成时事件驱动

数据源二：人工录入（后台操作人员）

#数据类型内容触发时机操作人
1发货登记序列号 + 客户姓名 + 电话 + 地址发货前仓库/平台
数据流向总结：设备上报的 6 种数据通过 WebSocket 实时进入云平台；人工录入的发货信息在设备上线前就已准备好。两者通过设备序列号关联。 目的是：让客户和设备关联，更全面的维护设备信息，这有助于我们平台的数据汇总和展示。
